# Contributing Guidelines

Welcome to the project! We’re excited to have you contribute.

## 📌 How to Contribute

1. Fork the repository
2. Create a feature branch: `git checkout -b feature/your-feature`
3. Commit your changes: `git commit -m "Add your feature"`
4. Push and submit a Pull Request

## 📄 Code Guidelines

* Follow [PEP8](https://www.python.org/dev/peps/pep-0008/) standards.
* Document your code with clear docstrings.
* Avoid adding any hardcoded credentials or sensitive information.

## 🧪 Testing

* All new features must include test coverage.
* Use `pytest` for unit testing if possible.

## 📜 Legal & Ethics

* All contributions must comply with our ethical standards and terms of use.
* Do not submit code that promotes or enables illegal activity.

Thank you for contributing to the future of ethical cybersecurity!
